# GPT-5.6 system correction

This note supersedes older global guidance where it conflicts with the rules below.

- Skill presence is not capability proof. Claim a route works only after a fresh-session
  trigger test or a live callable smoke test.
- Company H5 patterns, unified-admin sequencing, project facts, IDs, prices, payment
  rules, and Feishu operations apply only to matching company/work tasks. Generic,
  personal, client, and open-source work must use generic product/design/code routes.
- Generic H5 product design routes through Product Design context and ideation. Company
  H5 work uses analysis, H5 review, admin review, then PRD. Admin work must continue
  past the planning gate into the requested artifact instead of stopping after one step.
- PRD clarification is one compact batch of at most three material questions. Source
  inspection precedes drafting; routine sequencing does not require confirmation.
- Official Chrome and Computer Use are preferred for login-state browser work and
  Windows apps when live callable. Legacy browser, desktop, PDF, DOCX, spreadsheet,
  and image-to-code skills are fallbacks, not competing defaults.
- Open Design variants remain candidates except where an explicit comparison route
  selects one. Four-design-option requests must use structurally different lanes, not
  one method with recolors.
- Project-local duplicate skill copies are reference-only and cannot trigger implicitly.
- Automation prompts must point to project-local source-of-truth files and must not
  embed credentials, server addresses, or volatile business definitions.

Validation evidence from 2026-07-10:

- Full audit parsed 34 session files and 177,321 records with zero parse errors; indexed
  26 project roots, 40,752 project files, and 142 memory files.
- Skill integrity after fixes: 174 active-surface skill files, 174 unique names, zero
  exact duplicate names, zero conflicting duplicate names, and zero invalid frontmatter.
- Ten fresh `gpt-5.6-sol` route evaluations passed: generic H5, company H5, admin
  redesign, PRD source gate, four design lanes, Chrome login-state document, Windows
  Excel, data report, Feishu project sync, and public WeChat article.
- Official Computer Use listed Windows applications through the live client. Official
  Chrome returned live open tabs through the extension client. Lark CLI was updated to
  1.0.68 and its auth/docs/drive/whiteboard wrapper path was smoke-tested without Hermes.

Known residuals:

- Memory export mirrors create 43 exact duplicate groups. Treat
  `memories/extensions/ad_hoc/notes` as authoritative and mirrored copies as non-authoritative;
  physical deletion still requires explicit approval.
- External Open Design adaptations without upstream metadata are not update-verified.
- Multi-lane design and Feishu sync routes remain expensive, so select them only for
  explicit matching requests.
- VPN/remote compact disconnects and MCP startup/shutdown warnings are runtime/network
  issues, not fixed by skill routing changes.
