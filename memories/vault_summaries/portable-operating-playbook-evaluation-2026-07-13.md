# Portable Operating Playbook Evaluation

## User-Visible Defect

Another Codex instance could see that WeChat access work had happened, but the
export did not give it a primary, tool-agnostic operating instruction. The source
ledger mixed provenance, decisions, install state, and evidence, so an AI could
read it without learning the exact natural trigger, success condition, route
escalation, or denial scope.

## Baseline

- `README.md` told another AI to read `memories/self-evolution-ledger.md` before
  operational guidance.
- The ledger row said `agent-reach` and `web-access` existed, but did not itself
  explain a complete no-code behavior contract.
- The 2026-07-13 correction initially focused on installing and testing Camoufox.
  That fixed one Codex runtime, not the knowledge-transfer defect by itself.

## Adopted Correction

- Added `AI_OPERATING_PLAYBOOK.md` as the first cross-AI entry point.
- Defined the role of playbook, skill, ledger, evidence summary, script, and test.
- Added a complete public-WeChat capability card with natural triggers, body-level
  success criteria, tool-agnostic route ladder, route-scoped failures, stop rules,
  and reporting contract.
- Made local setup scripts optional implementations rather than prerequisites for
  understanding or applying the behavior.
- Changed `absorb-lessons`: a reusable capability cannot be marked fully adopted
  without a portable capability card and validation.

## Validation

- Structural evolution contracts: 42/42 passed.
- Export safety scan: 297 files scanned, 0 findings.
- Every skill path referenced by the core capability index exists: 9/9.
- Export and global `absorb-lessons` copies both passed skill validation.
- Independent fresh-model blind test: attempted through the official npm Codex
  CLI because the packaged desktop `codex.exe` returned `Access is denied`; npm
  retrieval produced no output and was terminated after approximately 80 seconds.
  This evidence tier remains `unverified` and is not reported as a pass.
- Prior live article extraction evidence remains separate: a fresh isolated reader
  environment extracted a 14,507-character body. That proves the optional helper,
  not by itself the no-code AI comprehension layer.

## Residual Risk

The playbook is now understandable without local code, but a genuinely independent
AI behavior test still needs to be run from another task/runtime that can read the
repository. The structural and live-tool evidence must not be relabeled as that
independent model test.
